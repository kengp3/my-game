## Phase Character

This phase is a **content replacement phase**, not a component-building phase. No new Vue SFC components, CSS classes, or design tokens are introduced. The implementation work is entirely authoring Markdown frontmatter and prose content within existing templates.

All design tokens, spacing, typography, color, and component contracts are inherited verbatim from Phase 2 UI-SPEC (`02-UI-SPEC.md`). This document records what changes, what stays the same, and the new copywriting patterns introduced by real game data.

## Interaction Contract

All interaction patterns are inherited from Phase 2. No new interaction states introduced.

### Placeholder Removal Contract

This phase's primary interaction change: the `:::warning 範例資料` marker is **removed** from all updated pages.

**Before (placeholder state):**
```
::: warning 範例資料
本頁內容為示範用佔位資料，尚未填入真實遊戲資訊。
:::
```

**After (real data state):**
- Marker removed entirely if all mandatory fields are populated
- If any field is missing, the page-level warning is still removed; missing fields use the field-level marker instead (D-07, D-08)

### Partial Data Marker (Field-Level)

For fields where real data is unavailable, the field-level info container is used:

```
::: info 資料待補充
此欄位資料尚未收集，待後續補充。
:::
```

This replaces the page-level warning for partially-complete pages. It is rendered using VitePress's built-in `:::info` container (blue background, info icon) — no custom CSS required.

### Table Hover (Inherited)

Mechanism: `.vp-doc table tr:hover` in `custom.css` — unchanged. All keybinding tables in CombatGuideTemplate and any data tables in new pages inherit this rule.

### Collapsible Spoiler Blocks (Inherited)

Mechanism: VitePress `:::details` container. Default collapsed state. Used in chapter pages for story summaries.

Label pattern (unchanged): `劇情摘要（點擊展開，含劇透）`

### Sidebar Navigation Update

New chapter pages (chapter-prologue, chapter-4 through chapter-12) must be added to `.vitepress/config.ts` sidebar under `/guide/story/`. Sidebar items use real chapter names in Traditional Chinese. No UI component changes — config.ts update only.

## Copywriting Contract

Phase 02.1 establishes new real-game copy and updates placeholder text to real content. All copy is in Traditional Chinese.

| Element | Copy |
|---------|------|
| Primary CTA (story pages — between chapters) | 前往下一章 (inherited) |
| Primary CTA (combat pages — to related guide) | 查看連招系統 (inherited) |
| Partial data field heading | 資料待補充 |
| Partial data field body | 此欄位資料尚未收集，待後續補充。 |
| Empty state — boss image | 圖片尚未提供 (inherited from BossTemplate) |
| Empty state — no drops data | 掉落物品資料尚未收集，請待後續更新。 |
| Empty state — no keybindings | 按鍵資訊尚未提供，敬請期待。 (inherited) |
| Spoiler toggle label (collapsed) | 劇情摘要（點擊展開，含劇透） (inherited) |
| Choice consequence (collapsed) | 查看後果 (inherited) |
| Recommended option badge | 推薦 (inherited) |
| Verified real data indicator | (no indicator needed — absence of `:::warning` signals real data) |

**Real content copy standards (new in Phase 02.1):**

Boss attack description pattern: `{動作名稱（英文原名）} — {動作描述，含前搖提示}`
Example: `泵踢（Pump Kick）— 雙腳飛踢，前搖為左膝舉起`

Chapter name pattern (sidebar + page title): `第{N}章：{繁中章節名稱}`
Example: `第一章：初次相遇`

Prologue follows a different pattern (no number): `序幕：黑夜突襲`

Area name pattern: `{繁中名稱}（{英文名稱}）` on first mention within a page; subsequent mentions use 繁中 name only.

Destructive actions in Phase 02.1: none. Wiki is read-only from the visitor's perspective.
