## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Fixed null boss.link crash in StoryChapterTemplate**
- **Found during:** Overall verification (VitePress build)
- **Issue:** `StoryChapterTemplate.vue` line 31 called `withBase(boss.link)` unconditionally. When `boss.link` is `null`, VitePress's `withBase()` throws `TypeError: Cannot read properties of null (reading 'startsWith')` at SSR render time, crashing the build.
- **Root cause:** The template was written assuming boss links would always be provided. Chapters 9-12 (and 2-8 from prior plans) have `link: null` on bosses where no dedicated page exists yet.
- **Fix:** Replaced `<a v-for="boss ...">` with `<template v-for>` containing `<a v-if="boss.link">` and `<span v-else class="boss-link-pending">`. Added `.boss-link-pending` CSS style (italic, muted text-2 color).
- **Files modified:** `.vitepress/theme/components/StoryChapterTemplate.vue`
- **Commit:** 40e9282
