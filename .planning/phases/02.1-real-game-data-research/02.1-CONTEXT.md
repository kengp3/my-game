# Phase 02.1: Real Game Data Research - Context

**Gathered:** 2026-04-10
**Status:** Ready for planning

<domain>
## Phase Boundary

從網路搜集赤血沙漠（Crimson Desert）真實遊戲資料，替換 Phase 1 和 Phase 2 建立的所有佔位內容頁面為真實資訊。包括戰鬥教學、主線劇情章節、Boss 範例、裝備範例、區域範例等現有頁面。如遊戲章節數超過現有 3 章，需新增對應頁面。

</domain>

<decisions>
## Implementation Decisions

### 搜集範圍與優先順序
- **D-01:** 覆蓋 Phase 1+2 所有現有內容頁面：戰鬥教學三頁（基礎操作、連招系統、防禦與格擋）、劇情章節（現有 3 章 + 依真實遊戲擴充）、Boss 範例頁、裝備範例頁、區域範例頁
- **D-02:** 劇情章節數量依真實遊戲章節數建立對應頁面，不限於現有 3 章

### 資料來源策略
- **D-03:** 廣泛搜集所有可用來源 — 官方網站、巴哈姆特、遊戲資料庫 (wiki)、Reddit、YouTube 等，交叉比對多個來源確保準確性
- **D-04:** 資訊衝突時由 Claude 根據資料類型和識別度自行判斷：數值型資料交叉比對，描述型資料取最詳細版本

### 內容深度與完整性
- **D-05:** 完整填滿模板所有 frontmatter 欄位和內容區塊 — Boss 攻擊模式表、裝備屬性數值、劇情步驟等全部到位
- **D-06:** 找不到資料的欄位使用統一的「資料待補充」標記（`::: info 資料待補充`）

### 佔位標記處理
- **D-07:** 替換完成的頁面移除所有 `::: warning 範例資料` 警告標記
- **D-08:** 部分欄位無法填充時，頁面級別的「範例資料」警告仍移除，改用欄位級別的「資料待補充」標記

### Claude's Discretion
- 內容優先順序 — 根據資料可取得性和欄位重要程度自行排序
- 衝突資料的裁定 — 根據來源可靠度和資料一致性判斷
- 新增章節頁面的 URL slug 命名
- 搜集工具和搜尋策略的選擇

</decisions>

- Vue SFC 模板使用 `useData()` 讀取 frontmatter
- Markdown 頁面使用 frontmatter + `<ComponentName />` 模式
