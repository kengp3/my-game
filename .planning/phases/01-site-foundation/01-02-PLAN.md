<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Frontmatter YAML -> Vue template | User-authored frontmatter rendered in Vue components via `{{ }}` interpolation |
| Markdown content -> HTML | Content in slot rendered through VitePress Markdown pipeline |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Spoofing | Vue template `{{ }}` interpolation | accept | Vue's `{{ }}` uses textContent (auto-escapes HTML). No v-html is used in templates. Frontmatter values are authored by trusted developers, not user input. XSS risk is negligible. |
| T-02-02 | Tampering | Frontmatter field injection | accept | All frontmatter is authored in Markdown files committed to git. No runtime user input reaches frontmatter. Version control provides audit trail. |
| T-02-03 | Denial of Service | Large frontmatter arrays | accept | Static site builds at deploy time, not per-request. Even very large arrays only affect build time, not user experience. No practical DoS vector. |
</threat_model>
