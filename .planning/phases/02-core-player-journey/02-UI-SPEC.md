## Spacing Scale

Declared values (multiples of 4 only). All values map to existing `.template-*` utility classes in `custom.css`.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, info-label margin-bottom, badge padding vertical |
| sm | 8px | Card gaps, badge padding horizontal, grid gap, inline element gap |
| md | 16px | Card padding (`.template-card`), default element spacing |
| lg | 24px | Section padding, choice-option spacing |
| xl | 32px | Section margin-bottom (`.template-section`) |
| 2xl | 48px | Section break margin (`.template-section-break`) |
| 3xl | 64px | Page-level top/bottom clearance |

Exceptions:
- Difficulty badge: 2px 8px padding (xs vertical, sm horizontal) — matches existing `.difficulty-badge` rule in `custom.css`
- Boss image placeholder height: 200px — inherited from BossTemplate, reused in CombatGuideTemplate image slot
- Touch targets for chapter nav buttons: minimum 44px height (mobile accessibility standard)

Source: `custom.css` utility classes (`.template-section`, `.template-section-break`, `.template-card`, `.difficulty-badge`) verified from codebase.

## Typography

All sizes use `Noto Sans TC` (400 regular, 700 bold). VitePress DefaultTheme manages heading hierarchy; these values extend the theme for new Phase 2 components.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 (regular) | 1.6 |
| Label | 14px | 700 (bold) | 1.4 |
| Info label (meta field) | 12px | 700 (bold) | 1.3 |
| Heading (h2 section) | 20px | 700 (bold) | 1.2 |

Rules:
- Body copy (推進路線, 劇情摘要) uses 16px / 400 / 1.6
- Labels inside `.template-card` (field names like "推薦等級", "涉及區域") use 12px / 700 / uppercase — matches existing `.info-label` rule in BossTemplate
- Values inside `.template-card` use 15px / 400 — matches existing `.info-value` rule
- Heading h2 rendered by VitePress DefaultTheme; no override required
- Difficulty/chapter number badge text: 14px / 700 — matches existing `.difficulty-badge` rule

Source: `.info-label` (12px, 700), `.info-value` (15px), `.difficulty-badge` (14px, 700) all verified in `custom.css` and `BossTemplate.vue`.

## Color

The full color palette is established from Phase 1 and must not be extended in Phase 2. New components use only the existing CSS variables.

| Role | Value (light) | Value (dark) | Usage |
|------|---------------|--------------|-------|
| Dominant (60%) | `#faf8f5` (`--vp-c-bg`) | `#1a1a1e` | Page background, main content area |
| Secondary (30%) | `#f0ece6` (`--vp-c-bg-soft`) | `#222228` | Cards (`.template-card`), sidebar, nav, info grids |
| Accent (10%) | `#a83520` (`--vp-c-brand-1`) | `#c5442a` | Reserved-for list below |
| Destructive | `#e54040` | `#e54040` | Destructive difficulty badge only (`data-level="極難"`) |

Accent (`--vp-c-brand-1`) reserved for:
1. Recommended choice highlight in ChoicePoint component (border-left or background tint using `--vp-c-brand-soft`)
2. Active sidebar link indicator (managed by VitePress DefaultTheme — no override)
3. CTA link text in "相關頁面" sections within CombatGuideTemplate
4. Chapter navigation button hover state (inherits VitePress brand link color)

Gold highlight (`#d4a843`, `.gold-highlight`) reserved for:
- Legendary rarity badge (inherited from Phase 1)
- Optional collectible hint callouts within chapter pages

No new color values are introduced in Phase 2. All new components use only the declared CSS variables.

Source: `custom.css` CSS variable declarations (light + dark modes) verified from codebase.

## Component Inventory

New components introduced in Phase 2. All follow the established Vue SFC + `useData()` + `withBase()` pattern from Phase 1.

### CombatGuideTemplate.vue

Page-level template. Reads frontmatter via `useData()`.

Rendered sections (in order):
1. Header — title + difficulty badge (`.difficulty-badge[data-level]`)
2. Prerequisites block — inline pill list (`.template-card` row)
3. Keybindings table — standard Markdown table rendered by `.vp-doc table`; headers: 操作、按鍵、說明
4. Tips list — ordered list with `--vp-c-brand-soft` left-border highlight strip
5. Related pages — horizontal link strip using `withBase()` links
6. Slot — Markdown body (詳細說明 prose content)

Frontmatter fields: `title`, `difficulty`, `prerequisites[]`, `keybindings[]{action, key, note}`, `tips[]`, `relatedPages[]{text, link}`

### StoryChapterTemplate.vue

Page-level template. Reads frontmatter via `useData()`.

Rendered sections (in order):
1. Chapter header — chapter number badge + chapter name + recommended level
2. Areas involved — comma-separated area chips (`.template-card` inline)
3. Key bosses — linked list using `withBase()`
4. Story summary — rendered using VitePress `:::details` container (NOT a custom component)
5. Choice points — rendered via embedded `<ChoicePoint>` calls, one per choice object in frontmatter `choices[]`
6. Slot — Markdown body (推進路線 prose content, collectible hints)
7. Chapter nav — VitePress auto-generated `prev`/`next` from frontmatter + sidebar order

Frontmatter fields: `title`, `chapterNumber`, `chapterName`, `recommendedLevel`, `areas[]`, `bosses[]{name, link}`, `choices[]{title, options[]{text, consequence, recommended}}`, `prev`, `next`

### ChoicePoint.vue

Inline component (not a page template). Accepts props, NOT frontmatter.

Rendered structure:
- Choice box container — 2px left border in `--vp-c-brand-1`; background `--vp-c-bg-soft`; padding 16px; border-radius 8px
- Title — 14px / 700, text "選擇點：{title}"
- Options list — each option: text label + consequence (collapsed behind native `<details>` toggle inside the component)
- Recommended option — highlighted with `--vp-c-brand-soft` background + "推薦" badge in `--vp-c-brand-1`

Props interface:
```typescript
interface Choice {
  text: string
  consequence: string
  recommended?: boolean
  link?: string
}
Props: { title?: string; choices: Choice[] }
```

Source: Architecture Patterns Pattern 4 from RESEARCH.md (A1 assumption, low risk).

## Interaction Contract

### Collapsible Spoiler Blocks

Mechanism: VitePress built-in `:::details` container — renders as native `<details>/<summary>` HTML.

Label pattern: `劇情摘要（點擊展開，含劇透）`

State behavior:
- Default state: collapsed
- Expanded: full prose content visible
- No animation required (browser default `<details>` behavior)

### Choice Consequence Reveal

Mechanism: Native `<details>` element inside ChoicePoint.vue (local Vue template, not VitePress container).

Label when collapsed: `查看後果`
Label when expanded: `收起後果`

### Chapter Navigation

Mechanism: VitePress DefaultTheme automatic prev/next generation from sidebar order.

Override allowed via frontmatter `prev`/`next` fields when sidebar order differs from reading order.

Button labels: inherited from VitePress DefaultTheme — "上一頁 / 下一頁" with custom text from frontmatter `text` field.

Touch target: minimum 44px height on mobile — VitePress DefaultTheme handles this natively.

### Placeholder Data Marker

All placeholder content pages display a VitePress `:::warning` custom block at top:

```
::: warning 範例資料
本頁內容為示範用佔位資料，尚未填入真實遊戲資訊。
:::
```

This is the single indicator that content is not real game data (D-07).

### Table Hover

Mechanism: `.vp-doc table tr:hover` rule in `custom.css` — `background-color: var(--vp-c-bg-soft)` with `transition: background-color 150ms ease`.

All keybinding tables in CombatGuideTemplate inherit this rule automatically.
