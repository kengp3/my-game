# Plan 03.1-04 Summary — Fix withBase() :src Binding in 3 Vue Templates

## Objective (delivered)

Fixed the `withBase()` bug in all 3 Vue SFC templates. Each now uses `:src="withBase(frontmatter.image)"` so the production GitHub Pages deploy (`base: '/my-game/'`) resolves image URLs correctly. Dimension 4 (Template Path Correctness) is GREEN.

## Root cause (recap from RESEARCH.md)

VitePress prepends `config.base` to asset URLs only when you route them through `withBase()`. Bare `:src="frontmatter.image"` worked in `docs:dev` (dev server does not apply base) but would 404 in production under `/my-game/`. All three templates already imported `withBase` for other bindings (`<a :href="withBase(...)">`) — only the `<img>` tag was missed.
