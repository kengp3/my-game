### Harness bug fix

`scripts/verify-images.mjs --dist` originally expected `.vitepress/dist/my-game/images/` but VitePress's `base: '/my-game/'` is a URL prefix applied at runtime, NOT a directory prefix in the dist tree. Actual layout is `.vitepress/dist/images/...`. Fixed in commit `722e511`:

- Updated `--dist` to look at `.vitepress/dist/images/` (actual build output)
- Added HTML base-prefix verification: reads each of the 8 target HTML files and confirms the rendered `<img src="/my-game/images/...">` URL contains the base prefix (proves `withBase()` was applied at render time)
- Added negative regression check: fails if any target HTML contains the bare `/images/...` src attribute (guards against withBase() regression)
