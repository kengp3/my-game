# Phase 03.1 Plan 01: Image Tooling Infrastructure Summary

Wave 0 infrastructure — installed sharp@0.34.5, created PNG-to-WebP batch converter and 5-flag Nyquist validation harness, created public/images directory tree.

## What Was Built

### sharp@0.34.5 installed (package.json devDependencies)

Exact version from package-lock.json: `0.34.5`. libvips backend confirmed loadable (`vips: 8.17.3`, `webp: 1.6.0`). Pinned via `^0.34.5` range + package-lock.json integrity hash (T-03.1-01 supply-chain mitigation).

### scripts/convert-images.cjs

Batch PNG-to-WebP converter with 8 hardcoded source→destination mappings:

| Source (/tmp/) | Destination | Max Width |
|----------------|-------------|-----------|
| kliff.png | public/images/characters/kliff.webp | 600px |
| damiane.png | public/images/characters/damiane.webp | 600px |
| oongka.png | public/images/characters/oongka.webp | 600px |
| kailok.png | public/images/bosses/example-boss.webp | 600px |
| sword-of-the-lord.png | public/images/equipment/example-equipment.webp | 400px |
| sword-of-the-wolf.png | public/images/equipment/sword-of-the-wolf.webp | 400px |
| grey-wolf-leather-armor.png | public/images/equipment/grey-wolf-leather-armor.webp | 400px |
| pailunese-signet.png | public/images/equipment/pailunese-signet.webp | 400px |

Key properties: quality=80, withoutEnlargement:true (prevents upscaling 300px sources to 600px), .cjs extension mandatory.

### scripts/verify-images.mjs — Flag Contract

| Flag | Dimension | What It Checks | Expected at Plan 01 Baseline |
|------|-----------|----------------|-------------------------------|
| `--files` | 1+2 | 8 WebP files exist, < 500KB, RIFF/WEBP magic bytes | EXIT 1 (files missing) |
| `--widths` | 2 | char/boss <= 600px, equipment <= 400px (uses sharp) | EXIT 1 (files missing) |
| `--frontmatter` | 3 | 8 .md files have `image: /images/{cat}/{slug}.webp` (not null) | EXIT 1 (8x IMAGE_NULL) |
| `--templates` | 4 | 3 Vue files use `withBase(frontmatter.image)`, 0 bare refs | EXIT 1 (3x BARE :src) |
| `--dist` | 5 | .vitepress/dist/my-game/images/*.webp after build | EXIT 1 (no dist yet) |

Exit 0 = PASS, Exit 1 = FAIL. Runs all flags when none specified.

### Pre-Wave-1 Harness Baseline (Verified)

```
node scripts/verify-images.mjs --frontmatter
  → EXIT 1, 8x IMAGE_NULL (all 8 target .md files still have image: null)

node scripts/verify-images.mjs --templates
  → EXIT 1, 3x BARE :src (CharacterTemplate, BossTemplate, EquipmentTemplate)
```

These non-zero exits **prove** the harness correctly detects the bugs it was designed to catch, before any downstream changes are applied.
