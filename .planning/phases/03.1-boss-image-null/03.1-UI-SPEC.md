## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#faf8f5` (light) / `#1a1a1e` (dark) | Page background (`--vp-c-bg`) |
| Secondary (30%) | `#f0ece6` (light) / `#222228` (dark) | Cards, sidebar, soft containers (`--vp-c-bg-soft`, `--vp-c-bg-alt`) |
| Accent (10%) | `#a83520` (light) / `#c5442a` (dark) | See reserved-for list below |
| Destructive | `#e54040` | Difficulty badge "極難" only (`data-level="極難"`) |

Source: `.vitepress/theme/custom.css` — full CSS variable mapping already declared.

**Accent reserved for:**
- Nav active link highlight
- Brand-colored interactive elements (VitePress default `--vp-c-brand-1`)
- Image alt text fallback placeholder border (new — Phase 03.1 specific: use `--vp-c-brand-soft` as placeholder background when `frontmatter.image` is null but component renders)

**Accent NOT used for:**
- Static text
- Table rows
- Section separators

**Phase 03.1 specific — image display context:**

Character and Boss images render at maximum 600px width within the `.template-card` context (16px padding each side). Equipment images render at maximum 400px width. Both use `object-fit: cover` behavior implied by the `<img>` tag with no CSS override — natural aspect ratio is preserved.

The `withBase()` fix changes no visual output but is required for correct URL resolution on GitHub Pages (`base: '/my-game/'`). Alt text uses the frontmatter `name` field in 繁體中文.
