## Spacing Scale

Same scale as Phase 1 and Phase 2. No new tokens introduced. All values map to existing `.template-*` utility classes in `custom.css`.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, info-label margin-bottom, badge padding vertical |
| sm | 8px | Card gaps, badge padding horizontal, grid gap, inline element gap |
| md | 16px | Card padding (`.template-card`), default element spacing |
| lg | 24px | Section padding, skill-row spacing |
| xl | 32px | Section margin-bottom (`.template-section`) |
| 2xl | 48px | Section break margin (`.template-section-break`) |
| 3xl | 64px | Page-level top/bottom clearance |

Exceptions:
- Rarity badge: 2px 8px padding (xs vertical, sm horizontal) — matches existing `.rarity-badge` rule in `custom.css`
- Character image and equipment image placeholders: 200px height — matches established BossTemplate and EquipmentTemplate placeholder height
- Skill table row: inherits `.vp-doc table` row height; no override needed
- Build recommendation card internal padding: 16px (md token) — matches existing `.template-card` padding

Source: `custom.css` utility classes verified from codebase.

## Typography

Same four-tier scale as Phase 2. No new font sizes may be introduced in Phase 3.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 (regular) | 1.6 |
| Label | 14px | 700 (bold) | 1.4 |
| Info label (meta field) | 12px | 700 (bold) | 1.3 |
| Heading (h2 section) | 20px | 700 (bold) | 1.2 |

Rules for Phase 3:
- Character and equipment page body prose uses 16px / 400 / 1.6
- Skill table headers ("技能名稱", "等級", "效果", "冷卻") use 14px / 700 — matches `.difficulty-badge` weight; rendered via `.vp-doc table th`
- Info labels inside `.template-card` (field names like "職業", "武器類型", "推薦等級") use 12px / 700 / uppercase — matches existing `.info-label` rule in BossTemplate
- Values inside `.template-card` (`.info-value`) use 16px / 400 — same pattern confirmed in Phase 2
- Build card name (`.build-name`) uses 15px / 700 — exception inherited directly from `EquipmentTemplate.vue` (`.build-name` scoped rule); this is a pre-existing exception, not a new size tier
- Build card description (`.build-description`) uses 14px / 400 — maps to label tier for secondary content; pre-existing in `EquipmentTemplate.vue`

Source: `custom.css`, `BossTemplate.vue`, `EquipmentTemplate.vue` verified from codebase.

## Component Inventory

### CharacterTemplate.vue (NEW — Phase 3)

Page-level template. Reads frontmatter via `useData()`. Follows the established `useData()` + `withBase()` + scoped styles pattern from BossTemplate and EquipmentTemplate.

Must be registered in `.vitepress/theme/index.ts` via `app.component('CharacterTemplate', CharacterTemplate)`.

**Primary focal point:** Character/class name (h1) + class-type badge — first rendered element, sets identity before all sections.

Rendered sections (in order):
1. Header — character name (h1) + class-type badge (styled like `.difficulty-badge` but using `--vp-c-brand-soft` background instead of difficulty colors)
2. Image — character portrait from `frontmatter.image`; falls back to placeholder div (200px height, `--vp-c-bg-soft` background, dashed `--vp-c-divider` border, "圖片尚未提供" text at 14px `--vp-c-text-3`)
3. Info grid — 3-column grid (`.boss-info-grid` pattern): 職業類型, 主武器類型, 推薦等級 — each cell is `.template-card` with `.info-label` / `.info-value`
4. Skills table — `frontmatter.skills[]` rendered as a table via `.vp-doc table`: columns 技能名稱 | 等級 | 效果 | 冷卻時間 | 備註
5. Build recommendations — `frontmatter.builds[]` rendered as `.template-card` stack (`.equipment-builds` pattern from EquipmentTemplate): each card shows build name (14px 700), description (14px 400), and comma-separated linked equipment list
6. Slot — Markdown body (additional strategy prose, playstyle notes)

Frontmatter schema:
```yaml
name: string           # Character/class name
classType: string      # 職業類型 (e.g. 劍士、弓手、法師)
mainWeapon: string     # 主武器類型
recommendedLevel: string | number
image: string | null   # Path relative to public/; null triggers placeholder
skills:
  - name: string
    level: number | string   # e.g. 1, "被動"
    effect: string
    cooldown: string | null  # e.g. "12秒", null for passives
    note: string | null
builds:
  - name: string
    description: string
    equipmentLinks:
      - text: string
        link: string   # withBase()-wrapped internal path
```

Responsive layout:
- Info grid: 3 columns on desktop (≥ 768px), 1 column stacked on mobile (< 768px) — same `@media (max-width: 768px)` breakpoint as BossTemplate
- Skills table: horizontal scroll on mobile via VitePress default `.vp-doc` table overflow behavior
- Build cards: single-column flex stack at all breakpoints

### EquipmentTemplate.vue (EXISTING — Phase 1, reused in Phase 3)

No structural changes. Reuse as-is per CONTEXT.md D-04 (locked decision).

Current sections: 屬性數值 table, 取得方式 sources list, 強化路線 ordered list, 適用建構 build cards, Markdown slot.

Phase 3 addition: the "適用建構" build cards will now include character page cross-links (via `build.characterLink` in frontmatter) — add optional `<a :href="withBase(build.characterLink)">` after the description if the field is present. This is a backward-compatible addition; existing pages without `characterLink` render unchanged.

### System Guide Pages (NEW — Phase 3, Markdown only)

Two new system guide pages under `guide/systems/`:
- `enhancement-system.md` — 強化系統完整流程
- `crafting-system.md` — 製作系統完整流程

These pages do NOT require a new Vue template. They use standard VitePress Markdown with VitePress custom containers (`:::tip`, `:::warning`, `:::info`) for callout blocks. No new component is introduced.

Layout pattern: Headings (h2 for major steps, h3 for sub-steps), numbered lists for sequential flow, tables for material/cost breakdowns, `::: info 資料待補充` for missing data.

## Interaction Contract

### Skill Table Row Hover

Mechanism: `.vp-doc table tr:hover` rule already in `custom.css` — `background-color: var(--vp-c-bg-soft)` with `transition: background-color 150ms ease`.

All skill tables in CharacterTemplate inherit this rule automatically.

### Build Card Equipment Links

Mechanism: `<a :href="withBase(link)">` inside the build card loop. Same-tab navigation. No external link indicator (these are internal cross-references).

Hover: text-decoration underline (VitePress `.vp-doc a:hover` default).

### Missing Data Marker

All missing game data uses the established Phase 2.1 pattern:

```
::: info 資料待補充
此技能/裝備的詳細資料尚未收集完整，將於後續更新補充。
:::
```

This is the single indicator for incomplete content. No other placeholder patterns are introduced.

### Equipment Source Cross-Links (CHAR-04)

Mechanism: `EquipmentTemplate.vue` `sources[]` array — each source has optional `link` field pointing to Boss page, map area page, or crafting system guide. Rendered via `<a v-if="source.link" :href="withBase(source.link)">`. This existing mechanism satisfies CHAR-04 without template changes.

### Collapsible Skill Details (optional)

If a skill has a long "效果" description, the prose detail goes into the Markdown slot section below the table. The table "效果" cell contains a short summary (≤ 40 characters). This avoids wide table cells on mobile — no new interaction component needed.

## Accessibility Baseline

Inherited from Phase 1 and Phase 2. No changes:
